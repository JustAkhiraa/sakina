✦ Guide de publication

Sakina sur le Google Play Store

Une PWA se publie sur Android via la technique TWA (Trusted Web Activity) : ton site web devient une vraie app installable, sans réécrire une ligne de code natif.

✓

Préparer le PWA Fait par moi

Manifest complété (icônes, catégories), politique de confidentialité rédigée et liée depuis les Réglages, service worker à jour.

1

Générer les icônes PNG À faire par toi — 2 min

Le Play Store exige des icônes PNG (le SVG seul ne suffit pas).

  1. Ouvre assets/icon-generator.html dans ton navigateur (double-clic, ça marche en file:// — pas de modules JS ici)
  2. Clique sur « Tout télécharger » — 4 fichiers PNG sont téléchargés
  3. Déplace-les depuis ton dossier Téléchargements vers assets/ du projet
Résultat attendu dans assets/ : icon-192.png, icon-512.png, icon-192-maskable.png, icon-512-maskable.png
2

Mettre Sakina en ligne (HTTPS) À faire par toi — 10 min

Le Play Store exige une URL publique en HTTPS. Avec GitHub Pages (gratuit) :

  1. Crée un compte sur github.com/signup si tu n'en as pas
  2. Crée un nouveau dépôt (bouton vert New), nomme-le par ex. sakina, laisse-le public
  3. Dis-moi quand c'est fait — je pousserai le code du projet dans ce dépôt avec toi
  4. Dans Settings → Pages du dépôt, source = branche main, dossier / (root)
  5. Ton app sera en ligne sur https://TON-PSEUDO.github.io/sakina/
Je peux t'accompagner à l'écran pour chaque clic si tu veux — dis-moi simplement quand tu as créé le compte et le dépôt vide.
3

Créer le compte développeur Google Play À faire par toi — 25 $, une fois

Rendez-vous sur play.google.com/console/signup, paie les 25 $ (frais uniques, à vie, pour toutes tes futures apps). Vérification d'identité possible sous 48h.

4

Générer le package Android (.aab) À faire par toi — 5 min

Va sur pwabuilder.com :

  1. Colle l'URL de ton site GitHub Pages (étape 2)
  2. PWABuilder analyse ton manifest et ton service worker — tout devrait être vert ✓
  3. Choisis Android → génère le package .aab signé
  4. Télécharge et garde précieusement le fichier signing.keystore généré : sans lui, impossible de publier une mise à jour un jour
  5. Note le nom de package (ex. com.tondomaine.sakina) et l'empreinte SHA-256 affichés — tu en as besoin à l'étape suivante
5

Vérifier le domaine Fichier préparé par moi à compléter par toi

Pour que l'app s'ouvre en plein écran sans barre d'adresse, Android doit vérifier que tu possèdes le site. J'ai créé .well-known/assetlinks.json — remplace-y les deux valeurs par celles obtenues à l'étape 4 :

"package_name": "com.tondomaine.sakina",
"sha256_cert_fingerprints": ["TON:EMPREINTE:SHA256:ICI"]

Puis republie sur GitHub Pages (redis-le-moi, je m'en occupe).

6

Publier sur Play Console À faire par toi — 20 min

  1. Dans Play Console, crée une nouvelle application
  2. Upload le fichier .aab de l'étape 4 dans Production → Créer une release
  3. Renseigne la fiche : description, catégorie (Style de vie ou Éducation), icône 512×512 (déjà générée), 2 captures d'écran minimum (fais-en depuis ton téléphone une fois l'app testée)
  4. Colle l'URL de privacy-policy.html dans le champ Politique de confidentialité
  5. Complète le questionnaire de classification du contenu (public général, pas de contenu sensible)
  6. Choisis Gratuite, valide, envoie en revue — compte 1 à 7 jours pour l'approbation Google
