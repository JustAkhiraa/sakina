✦ Documentation du projet

Sakina

سَكِينَة

Compagnon spirituel tout-en-un : tasbih, horaires de prière, qibla, Coran (tajwid, favoris, notes), invocations, et outils (Qadâ', Rak'ah, Zakat, calendrier hégirien & du jeûne). PWA installable, hors-ligne, sans publicité.

« Sakina : la quiétude que Dieu fait descendre dans les cœurs — l'émotion que l'app cherche à susciter. »
● PWA installable ● Hors-ligne Vanilla JS · ES Modules Sans publicité

▶Lancer l'app

Les modules ES exigent un serveur HTTP — impossible d'ouvrir index.html en double-clic (protocole file:// bloqué par le navigateur).

python -m http.server 8642
# puis ouvrir http://localhost:8642
💡
Le service worker (mode hors-ligne) ne s'active qu'en HTTPS — déployez sur GitHub Pages, Netlify ou Vercel pour la PWA complète et installable.

3Architecture

index.html Coquille HTML (pages + sheets), zéro logique manifest.webmanifest PWA (installable, standalone) sw.js Offline : shell cache-first, API Coran réseau→cache css/ tokens.css Thème sombre/clair/nuit OLED, 8 accents base.css Reset, glassmorphism, composants partagés pages.css Styles par écran js/ app.js Point d'entrée : câblage des modules core/ store.js État persistant + migration auto (tpv5) router.js Navigation par pages + hooks d'affichage ui.js Toast, burst, sheets, confirmation (Promise) audio.js Sons WebAudio synthétisés + haptique lib/ astro.js Horaires de prière, qibla, distance Ka'ba hijri.js Calendrier hégirien tabulaire data/ Dhikrs, douas, 114 sourates, méthodes de calcul features/ Un module par écran (tasbih, salat, qibla…) legacy/v8.html Ancien monolithe (référence, ne plus modifier)

Communication inter-modules

Imports directs quand la dépendance est naturelle, CustomEvent DOM (stats-changed, location-changed) pour éviter les cycles.

Données utilisateur

localStorage clé sakina.v1. Au premier lancement, les données de l'ancienne app (tpv5 + qdata) sont migrées automatiquement. Écritures groupées (80 ms) avec sauvegarde garantie sur fermeture/mise en arrière-plan.
